# SEO Battle Plan: Dominating the Danish-Spanish Legal Services Market

## Executive Summary

This plan outlines a comprehensive strategy to outrank and capture traffic from three key competitors:
- **Zafolaw.com** (Primary competitor - Spanish law for Scandinavians)
- **Lindkaerlaw.dk** (Danish law firm with international practice)
- **Gorrissenfederspiel.com** (Large corporate firm with Spanish desk)

**Goal:** Become the #1 search result for Danish users seeking Spanish legal services.

---

## Competitor Analysis

### 1. Zafolaw.com - PRIMARY COMPETITOR

| Aspect | Analysis |
|--------|----------|
| **Positioning** | "Friend of Scandinavia" - Spanish law firm serving Nordic clients |
| **Languages** | Danish, Swedish, English, Spanish |
| **Content Volume** | 30+ articles, 15+ downloadable PDFs |
| **Strengths** | Multi-language, B2B focus, established brand, webinars, newsletter |
| **Weaknesses** | Generic corporate feel, large firm = less personal, slower response times |
| **Target Keywords** | "spansk selvangivelse", "fast ejendom spanien", "NIE-nummer", "ansættelseskontrakter spanien" |

**Key Vulnerability:** They're a large firm based in Spain. You can position as the Danish-based alternative with faster response times and personal service.

### 2. Lindkaerlaw.dk - SECONDARY COMPETITOR

| Aspect | Analysis |
|--------|----------|
| **Positioning** | Specialized Danish law firm (Copenhagen-based) |
| **Languages** | Danish, English |
| **Content Volume** | Limited blog content |
| **Strengths** | Strong Danish legal credentials, litigation focus |
| **Weaknesses** | Not specialized in Spanish law, limited Spain-specific content |
| **Target Keywords** | "erhvervsret", "strafferet", "udlændingeret", "arbejdsret" |

**Key Vulnerability:** General Danish law focus means they can't compete on Spanish law depth. Capture their cross-border referrals.

### 3. Gorrissenfederspiel.com - TERTIARY COMPETITOR

| Aspect | Analysis |
|--------|----------|
| **Positioning** | Full-service international corporate law firm |
| **Languages** | Danish, English |
| **Content Volume** | High - news, publications, events |
| **Strengths** | Brand prestige, corporate clients, Spanish desk |
| **Weaknesses** | Premium pricing, B2B only, intimidating for individuals |
| **Target Keywords** | "internationalt advokatfirma", "M&A", "corporate law" |

**Key Vulnerability:** They serve corporations, not individuals. Capture all B2C Spanish legal searches.

## Technical SEO Checklist

### On-Page Optimization

- [ ] **Title tags**: Include primary keyword + location + brand
  - Format: `[Primary Keyword] | [Secondary Keyword] | Christian Gefke`
  - Example: `Køb Bolig Spanien | Dansk Advokat | Christian Gefke`

- [ ] **Meta descriptions**: Action-oriented, 150-160 characters
  - Include: Primary keyword, value proposition, call-to-action

- [ ] **H1 tags**: One per page, exact match or close variant of target keyword

- [ ] **URL structure**: Short, keyword-rich, no dates
  - Good: `/kob-bolig-spanien`
  - Bad: `/2024/01/15/hvordan-kober-man-bolig-i-spanien`

- [ ] **Internal linking**: Every page links to 3-5 related pages
  - Service pages link to relevant blog posts
  - Blog posts link to service pages
  - Create hub pages for each topic cluster

- [ ] **Image optimization**:
  - Descriptive alt tags with keywords
  - Compress images (WebP format)
  - Lazy loading enabled

### Schema Markup Implementation

Add structured data for:

```json
{
  "@type": "LegalService",
  "name": "Christian Gefke - Spansk Advokat",
  "areaServed": ["Denmark", "Spain"],
  "serviceType": ["Real Estate Law", "Tax Law", "Immigration Law", "Inheritance Law"]
}
```

Additional schemas:
- [ ] **FAQPage** - All FAQ sections
- [ ] **Article** - All blog posts
- [ ] **BreadcrumbList** - All pages
- [ ] **LocalBusiness** - Contact page
- [ ] **Person** - About page (author credentials)
- [ ] **Review** - Testimonials page

### Site Speed & Core Web Vitals

Current issues to address:
- [ ] Implement proper image lazy loading
- [ ] Minimize JavaScript bundle size
- [ ] Enable browser caching
- [ ] Use CDN for static assets
- [ ] Target LCP < 2.5s, FID < 100ms, CLS < 0.1
